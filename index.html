📘 Read Together · Share & Login 模块

面向家长 × 儿童的阅读场景，设计分享与登录两大核心模块，在 COPPA / CCPA 约束下 同时兼顾孩子成就反馈、家长记录成长与产品增长路径。

  1. 重新定义儿童阅读场景下的分享问题框架：

    解决“孩子需要成就反馈”“家长需要记录成长”“App 需要自然增长路径”三重需求冲突， 同时应对 COPPA 对儿童隐私的严格限制。最终提出分享不是炫耀，而是“孩子成就 → 家长记录 → 产品增长”的家庭闭环。

  2. 构建 Parent–Child–Platform 的三方心理模型：

    梳理家长记录需求、孩子成就表达需求与平台增长诉求，明确分享主操作者为家长、孩子只接收激励反馈， 形成适用于 Family-tech 产品的目标心智模型。

  3. 设计儿童阅读场景下的分享触发体系：

    按阅读完成、章节完成、连续天数、奖章达成等节点设计触发规则，将分享行为嵌入“正向反馈时刻”， 提升自然分享率与家庭参与度。

  4. 构建自动化内容生成与卡片模板体系：

    设计内容抽取规则（阅读时长、书籍封面、成就类型等），建立一键生成的分享卡片模板， 保证家长无需编辑即可分享，同时不包含儿童敏感信息以满足 COPPA 合规。

  5. 设计多平台分享策略与路径（美区社交生态）：

    分析 Facebook / Messenger / Instagram / WhatsApp 等在家庭场景下的使用差异， 构建 “Export-only → Private family share → Public-safe share” 的分层路径，在隐私安全前提下支持产品增长。

  6. 设计登录策略与底层交互逻辑：

    搭建“家长唯一操作者、孩子不参与登录”的策略框架，设计 Apple / Email / SMS 三种认证方式及两步极简流程， 并完成验证码、倒计时、异常处理、按钮状态机、会话管理与埋点 / KPI 指标的完整设计。

👗 Gensmo · AI 试穿产品（QA 测试）

围绕 avatar 生成、AI 换装与内容浏览链路，负责 iOS / Android 多端测试用例设计与回归测试， 确保生成效果与交互流程在真实场景下稳定可用。

  1. 设计并构建多模块测试用例体系：

    覆盖 Avatar（虚拟形象生成）、Try-On（AI 换装）、Feed 浏览、Share 分享、Remix 重装扮等模块， 保证不同功能、路径与输入组合都具备标准化用例。

  2. 执行跨端（iOS / Android）功能测试与回归测试：

    按版本迭代节奏验证自拍上传、模型生成、结果展示、交互操作等流程的正确性与稳定性， 发现问题后跟进修复并再次回归测试。

  3. 设计并覆盖异常与边界场景：

    包含网络不稳定、上传失败、生成超时、权限拒绝、用户中断等情况，为这些路径设计合理提示、 fallback 策略与状态恢复逻辑，提升整体鲁棒性。

  4. 验证多端一致性与交互体验一致性：

    比对 iOS / Android 在 UI 展示、生成速度、渲染质量、权限与资源加载上的差异， 输出多端一致性问题清单与优化建议，保证跨平台体验统一。

  5. 输出测试报告与缺陷分析并参与上线评估：

    整理 BUG 分类、严重等级、复现条件与影响范围，配合产品与研发评估版本质量， 为是否上线提供质量依据。

🧭 优势星球 · 用户访谈与需求洞察

面向「引流 → 测评 → 深度咨询 → 服务升级」的长链路服务场景，参与企业级用户访谈、 服务蓝图与用户旅程梳理，帮助识别关键痛点与真实需求。

  1. 负责访谈前的准备工作与样本设计：

    按照业务链路梳理访谈目标与问题框架，结合运营 / 销售 / 教练 / 用户等角色筛选访谈对象， 构建覆盖关键角色的访谈样本结构。

  2. 参与企业级用户访谈并记录真实流程：

    协助 mentor 开展深度访谈，追踪测评、咨询与转化各阶段的操作方式、等待节点与体验痛点， 获取与文档不同的一手流程信息。

  3. 整理访谈记录并沉淀用户旅程：

    将零散发言结构化为信息点、情绪变化与行为路径，补全 User Journey 中的关键阶段、 断点与决策点，为后续诊断提供基础。

  4. 绘制服务蓝图（Service Blueprint）：

    区分前台行为、后台支撑与系统流程，标记影响转化效率的关键触点，为后续流程优化与工具设计提供依据。

  5. 协助判断真实需求与伪需求：

    结合访谈内容与业务目标，识别表面抱怨背后的真正决策因素， 帮助团队聚焦在等待时间、反馈即时性、咨询价值感等高影响需求上。

2022.09 – 2022.11 市场部实习

南京宜家 · 市场部实习生

在宜家高流量零售环境中参与日常运营，包括商品陈列巡检、补货记录、动线观察与缺货分析； 支持市场部保持货架完整性、优化顾客体验，并保障高峰期运营稳定。

  1. 负责商场重点区域的商品陈列检查与补货记录：

    定期巡检高流转区域、促销区与样板间，记录缺货与低库存商品，确保陈列完整度； 通过及时发现断货风险保证销售连续性。

  2. 建立每日补货清单并与仓库团队对接：

    根据巡检结果整理补货清单，标注商品编号、数量变化与陈列优先级， 协助仓库完成每日补货排程，提高补货效率。

  3. 对缺货商品进行原因分类与初步分析：

    将缺货情况按“高流量消耗”“后台补货滞后”“展示损耗”等原因分类， 帮助市场团队定位库存周转问题，提升整体陈列策略的准确性。

  4. 监测动线区域的商品暴露度与顾客接触点：

    观察顾客在重点区域的停留行为、拿取频次与拥堵情况，记录陈列位置对产品曝光与购买率的影响， 为陈列优化决策提供现场依据。

  5. 完成每日现场记录、数据同步与例会汇报：

    将巡检结果整理成日报，并同步给市场团队，支持补货、陈列与运营人员进行快速决策， 保证商场在不同客流情况下保持稳定运营。
